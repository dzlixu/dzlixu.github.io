html 页面布局

发表于 2018-03-15

页面布局是把整个网页中要显示的内容，按结构和样式特点，划分成结构明显的若干个块，在实现网页样式时便于从整体到局部实现。开始学习前端时，对布局不太重视，工作中才发现，拿到设计图后能快速拿出可行的布局方案即是能力的体现，也为后面一步步实现打下基础。准确可行的布局方案，可以加快设计图的实现，可以节省设计图调整后的代码修改量。

元素在页面中占据的空间

在做页面布局前，你的理解css 页面元素盒子模型，html文档的渲染，把元素看作一个盒子，盒子有内容的宽高，内容到边框四周的距离（内边距），边框的宽度，以及边框以外四周的宽度。
一个元素的实际尺寸：元素的实际宽度 = 左右外边距 + 左右边框 + 左右内边距 + width;
元素的实际高度 = 上下外边距 + 上下边框 + 上下内边距 + height;

元素定位
元素在页面中的位置，需要给页面定位。定位的方式有5种，分别是普通文档流定位，浮动定位，相对定位、绝对定位、固定定位。

普通文档流定位
页面中的元素，按顺序在页面中依次显示，其中块级元素（div p h1-h6 ul ol table）会独占一行，行内元素和行内块元素

浮动定位
使块级元素能够在一行内显示 ，浮动定位的元素脱离了普通文档流，所以浮动元素的父元素的高度会变为0，浮动让行内元素成为快元素。

相对定位 relative
是用相对定位的元素会相对于它原来的位置，偏移某个距离。使用的场景有 3个：要实现元素本身位置的微调时使用；配合弹出菜单的子元素做相对定位；显示调整当前元素的堆叠顺序；

绝对定位 absolute
绝对定位的元素会脱离普通文档流，不占据页面空间。绝对定位的初始化位置相对于最近的”已定位”的”祖先元素”来进行位置的初始化。如果没有已经定位的父祖先元素，初试位置相对于body。

固定定位 fixed
绝对定位的位置是相对于浏览器窗口。固定定位的元素推理普通文档流，不占页面空间。
